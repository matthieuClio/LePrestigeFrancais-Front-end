# Features structure

Every features must adhere this structure: 

- Components/ → Feature-specific UI  
- Pages/ → routing pages (the page that will be called by the router)  
- Types/ → TypeScript types  
- Api/ → backend call (if necessary)  
- Hooks/ → business logic (if necessary)

⚠️ Create a folder only if it's necessary

# Shared folder
Shared folder will contain chared components

# Styles folder
Style folder will contain global style

# Tests folder
Tests folder is here where we will write our tests
